# 01_DevWorkspace

统一开发工作区根目录，用于集中管理工作项目、开源项目、个人项目、实验、工具、文档与归档内容。

## 目录说明

- `01_Work/`：所有工作相关内容，包括代码仓库、工作文档、脚本和临时联调材料
- `02_OpenSource/`：开源参与与自有开源项目
- `03_Personal/`：个人长期维护项目
- `04_Labs/`：非正式实验、PoC、探索性研究
- `05_Templates/`：项目模板与脚手架
- `06_Tools/`：非工作专属的通用开发工具与环境配置
- `07_Docs/`：非工作专属的通用技术文档与笔记
- `08_Archive/`：已归档项目
- `09_Scratch/`：非工作专属的随手测试与小片段代码

## 当前工作目录约定

当前工作内容统一收敛在 `01_Work/`，不再额外增加 `KM/` 这一层。

```text
01_Work/
├─lowcode/
├─mom/
├─docs/
├─tools/
└─temp/
```

- `lowcode/`：低代码相关正式仓库
- `mom/`：MOM 相关正式仓库
- `docs/`：工作方案、路线图、会议纪要、发布说明等
- `tools/`：工作脚本、SQL、环境辅助文件等
- `temp/`：临时联调文件、导出文件、一次性脚本等

## 使用约定

1. `D:\01_DevWorkspace` 本身不作为总 Git 仓库。
2. 每个具体项目单独初始化并维护自己的 Git 仓库。
3. 所有工作相关内容统一放入 `01_Work/`，不要分散到 `06_Tools/`、`07_Docs/`、`09_Scratch/`。
4. 实验成熟后迁移到正式分类目录。
5. 停止维护的项目移动到 `08_Archive/`。
